# TOF-MS データ解析ツール

飛行時間型質量分析計（TOF-MS）の測定データをブラウザ上で読み込み、可視化・解析するためのツールです。サーバーへのデータ送信は一切行わず、すべての処理はブラウザ内（クライアントサイドJavaScript）で完結します。

**公開ページ:** https://frmcle-m.github.io/tof-ms-tool/

## 1. 概要

- 入力は「測定データ（`.csv`）」と「測定条件（`.887`）」のペアファイルです。同じファイル名（拡張子違い）を1組として自動認識します。
- 読み込んだデータ・ラベル・表示設定はブラウザのIndexedDBに自動保存され、次回同じページを開いた際に復元されます。外部サーバーへの送信は行われません。
- グラフ描画には [Plotly.js](https://plotly.com/javascript/) を使用しています（CDN経由で読み込み）。
- 実装は `index.html`（画面構造）、`style.css`（見た目）、`app.js`（処理ロジック）の3ファイルに分かれています。

## 2. 入力ファイル形式

### 2.1 `.csv`（測定データ）

タブ区切り（区切りが見つからない場合は空白区切りにフォールバック）で、1行に「時間 (ns)」と「カウント数」の2列を持つテキストファイルです。

```
0.000   12
0.100   15
0.200   9
...
```

数値として解釈できない行は無視されます。

### 2.2 `.887`（測定条件）

`キー = 値` または `キー: 値`、あるいはスペース区切りの `キー 値` の形式で1行ずつ書かれた測定条件ファイルです。`REPORT-FILE from ... written` の形式の行からは測定時間を抽出します。ツールが認識・表示する主なキーは以下の通りです。

| キー | 表示名 |
|---|---|
| `REALTIME` | Real time (s) |
| `TOTALSUM` | Total counts |
| `SWEEPS` | Sweeps |
| `swpreset` | Sweep preset |
| `range` | Range (ch) |
| `calunit` | Cal. unit |
| `calfact` | Cal. factor |
| `caloff` | Cal. offset |
| `syncout` | Sync out (Hz) |

`caloff` の値は時間軸のオフセット補正に使用されます（`calfact` は二重変換を避けるため時間軸補正には使用しません）。

### 2.3 ファイル名と日付グルーピング

ファイル名の先頭が `YYYYMMDD` （4〜8桁の数字＋`-`または`_`）で始まる場合、その日付をキーとしてファイルを自動的にグループ化します。マッチしないファイルは「(日付不明)」グループにまとめられます。日付タブを切り替えると、一覧・ラベル付け・グラフの対象がその日付のファイルのみに絞られます（他の日付のファイルも読み込んだ状態のまま保持されます）。

## 3. 画面構成と機能

### 3.1 ファイルを入れる

- ドラッグ&ドロップ、またはファイル選択ボタンで `.csv`/`.887` ファイルをまとめて読み込みます。
- 「すべてクリア」で読み込んだデータとIndexedDBの保存内容を削除します。

### 3.2 日付を選択

- ファイル名から抽出した日付ごとにタブ表示し、以降のセクション（一覧・ラベル・グラフ）の対象を絞り込みます。

### 3.3 各ファイルの名前を見る

- 読み込んだファイルの一覧を表で表示します。列は「ファイル名」「.csvの有無」「.887の有無」「測定時間」「SWEEPS」です。

### 3.4 各ファイルにラベルをつける

- 各ファイルに数値または文字列のラベルを付与します。数値ラベルは複数ファイルグラフ（重ね書き・積算プロット）の横軸として使用できます。
- ラベルの種別（数値/文字列）・`.887`ファイルの`calfact`値を確認できます。
- 各ファイルについて「プロットに使用するか」「BG（バックグラウンド）として使用するか」をチェックボックスで指定します。BGに指定したファイルは差分・重ね書び・積算の各グラフでバックグラウンドとして扱われます。

### 3.5 グラフを描画する

グラフ描画エリアの幅・高さ（px）は共通設定として指定でき、PNG保存時のサイズにも反映されます。ファイル名がグラフ幅より長い場合は自動的に改行してタイトル等に表示します。5つのタブに分かれています。

幅・高さの入力欄の隣にある「⚙ 表示設定」ボタンを押すと、以下の表示条件を共通設定として変更できるパネルが開閉します（グラフサイズと同様、すべてのタブの描画・PNG保存に反映されます）。

- **データ線の色**: オンにすると、1本の線だけのグラフ（TOFスペクトル・質量スペクトル・差分など）に指定した色を使います。オフの場合は各グラフの既定色（steelblue・crimsonなど）のまま表示します。
- **カラーパレット**: 差分（Signal/BG）や複数ファイル重ね書きなど、1つのグラフに複数の線がある場合の配色です。「既定」以外（Category10・Set1・Set2・Pastel・モノトーン(青系)）を選ぶと、その配色で系列を塗り分けます。
- **線の太さ (px)**: 各グラフのデータ線の太さです。
- **凡例をグラフ内に表示する（inset）**: オフの場合は現在と同じくグラフの外側（右）に凡例を表示します。オンにするとグラフ内側（右上）に重ねて表示します。

#### (1) TOFスペクトル

選択した1ファイルの生データ（時間 vs カウント数）を表示します。測定条件の表示、グリッド表示、縦軸対数スケール、横軸・縦軸の範囲（自動/手動）を切り替えられます。

#### (2) 質量校正・質量スペクトル

TOFの時間軸を質量軸（u）に変換するための質量校正と、質量スペクトルの表示を行います。

- **ピーク検出**: 指定した検出閾値（counts）以上、かつ指定した最小ピーク間隔（channel）を満たす極大点をピークとして検出します（`scipy.signal.find_peaks` 相当のアルゴリズム）。
- **質量校正**: 検出したピークが、指定した質量ステップ（u、例: Heクラスターなら4.0026u）の整数倍間隔で並んでいると仮定し、開始質量番号 n=1〜50 の範囲で最小二乗直線回帰（`t = a√(mass) + b`）を総当たりし、決定係数(R²)が最も高い校正パラメータ `a`, `b` を採用します。校正ピーク検出範囲を時間軸で限定することもできます。
- 校正で得られた `a`, `b` は「差分」「複数ファイル重ね書き」タブの質量軸換算パラメータに自動反映されます。
- **ヤコビアン補正**: 時間軸から質量軸への変換で生じる間隔の不均一さを補正するオプションです（既定でON）。
- 質量スペクトルはスムージング（移動平均）を適用したうえで表示でき、質量ステップの整数倍位置に補助線、校正に使ったピーク位置に印を表示できます。横軸(u)・縦軸の範囲指定、縦軸対数スケールに対応します。

#### (3) 差分 (Signal − BG)

SignalファイルからBackgroundファイルを差し引いた差分スペクトルを表示します。横軸は時間(ns)または質量(u)（質量校正タブのa, bパラメータを使用）を選択できます。スムージング窓幅、測定条件表示、縦軸対数スケール、軸範囲の指定に対応します。

#### (4) 複数ファイル重ね書き

複数の信号ファイル（数値ラベルが付与されたファイルのみ選択可）を、共通のBGファイルを差し引いたうえで1つのグラフに重ねて表示します。ラベル値の昇順に並び替えて描画されます。横軸は時間(ns)または質量(u)を選択可能、グリッド表示・縦軸対数スケール・軸範囲指定に対応します。

#### (5) 積算・比プロット

指定した2つの時間範囲（範囲1・範囲2）内のカウントをそれぞれ積算し、ファイルごとの「範囲1」「範囲2」「範囲1/範囲2の比」を計算して、ラベル値を横軸とした散布図・折れ線グラフとして表示します。結果は表としても一覧表示されます。

- 対象ファイルは数値ラベルが付与されたファイルのみ選択可能です。
- 同一ラベル値を持つ複数ファイルがある場合、標準偏差などのばらつきを誤差棒として表示できます。
- データ点を線で結ぶか点のみにするか、横軸・縦軸それぞれの対数スケール表示に対応します。

### 3.6 グラフの保存・コピー

各グラフ画像の右上にカーソルを合わせると表示されるツールバー（モードバー）から、PNG画像として保存・コピーできます。

- **カメラのアイコン**: 表示中のグラフを、「⚙ 表示設定」の隣にあるグラフサイズで指定した幅・高さのPNG画像としてダウンロードします。
- **コピーのアイコン**（カメラのアイコンの隣）: 表示中のグラフをPNG画像としてクリップボードにコピーします。保存せずにそのままWord・PowerPoint・Slackなどに貼り付けて使えます。

## 4. データの永続化とプライバシー

- 読み込んだファイルの内容（パース済みデータ）・ラベル・BG指定・チェックリストの選択状態は、ブラウザのIndexedDBに自動保存されます。
- 次回同じブラウザで同じページを開いた際、保存内容から状態を復元します（別ブラウザ・別端末には引き継がれません）。
- データが外部サーバーに送信されることはありません（Plotly.jsのCDN読み込みを除き、すべてローカルで処理されます）。

## 5. 動作環境

- モダンブラウザ（Chrome, Edge, Safari, Firefoxなど）で動作します。IndexedDBが利用できない環境では、データ保存機能なしで動作を継続します。
- インターネット接続が必要です（Plotly.jsをCDNから読み込むため）。
