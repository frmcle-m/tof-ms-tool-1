# TOF-MS データ解析ツール

飛行時間型質量分析計（TOF-MS）の測定データをブラウザ上で読み込み、可視化・解析するためのツールです。サーバーへのデータ送信は一切行わず、すべての処理はブラウザ内（クライアントサイドJavaScript）で完結します。PCでの利用を前提としています。

## 1. 概要

- 入力は「測定データ（`.csv`）」と「測定条件（`.887`）」のペアファイルです。同じファイル名（拡張子違い）を1組として自動認識します。
- 読み込んだデータ・ラベル・BG設定・表示設定（軸範囲や質量校正の入力値など）はブラウザのIndexedDBに自動保存され、次回同じページを開いた際に復元されます。外部サーバーへの送信は行われません。「すべてクリア」を押すと、読み込んだファイルとあわせて、保存されている表示設定・校正値などの解析結果もすべて削除されます。
- グラフ描画には [Plotly.js](https://plotly.com/javascript/) を使用しています（CDN経由で読み込み）。
- 実装は `index.html`（画面構造）、`style.css`（見た目）、`app.js`（解析ロジック・描画処理）の3ファイルに分かれています。`app.js` の内容は、ツール画面のヘッダーおよびフッターにある「コードを編集する（app.js）」リンクから、別ページ（`editor.html`）で確認・編集できます。

## 2. 入力ファイル形式

### 2.1 `.csv`（測定データ）

タブ区切り（区切りが見つからない場合は空白区切りにフォールバック）で、1行に「時間 (ns)」と「カウント数」の2列を持つテキストファイルです。

```
0.000   12
0.100   15
0.200   9
...
```

数値として解釈できない行は無視されます。

### 2.2 `.887`（測定条件）

`キー = 値` または `キー: 値`、あるいはスペース区切りの `キー 値` の形式で1行ずつ書かれた測定条件ファイルです。`REPORT-FILE from ... written` の形式の行からは測定時間を抽出します。ツールが認識・表示する主なキーは以下の通りです。

| キー | 表示名 |
|---|---|
| `REALTIME` | Real time (s) |
| `TOTALSUM` | Total counts |
| `SWEEPS` | Sweeps |
| `swpreset` | Sweep preset |
| `range` | Range (ch) |
| `calunit` | Cal. unit |
| `calfact` | Cal. factor |
| `caloff` | Cal. offset |
| `syncout` | Sync out (Hz) |

`caloff` の値は時間軸のオフセット補正に使用されます（`calfact` は二重変換を避けるため時間軸補正には使用しません）。画面上に `calfact` を表示する箇所（ファイル一覧の表・測定条件の表示など）では、整数値として小数点以下を切り捨てて表示します。

### 2.3 ファイル名と日付グルーピング

ファイル名の先頭が `YYYYMMDD` （4〜8桁の数字＋`-`または`_`）で始まる場合、その日付をキーとしてファイルを自動的にグループ化します。マッチしないファイルは「(日付不明)」グループにまとめられます。

## 3. 画面構成

画面左側には常時、目次（index）が表示されます。「1. ファイルを入れる」「2. 各ファイルの一覧・ラベル」「3. グラフを描画する」（グラフはさらに5つのタブへ個別にリンク）への直接リンクに加え、**🗑 ゴミ箱**（別ページへのリンク。ゴミ箱に入っているファイル数を件数バッジで表示）、そして**これまでに読み込んだ全ての日付の履歴**が一覧表示されます。日付は複数選択でき、選んだ日付のファイルが「2. 各ファイルの一覧・ラベル」にまとめて表示されます。

### 3.1 ファイルを入れる

- ドラッグ&ドロップ、またはファイル選択ボタンで `.csv`/`.887` ファイルをまとめて読み込みます。
- 新しく読み込んだファイルの日付は、サイドバーの日付一覧に自動的に追加され、選択状態になります。
- 「すべてクリア」で読み込んだデータとIndexedDBの保存内容（表示設定・校正値・ラベル・BG設定を含む）を削除します。

### 3.2 各ファイルの一覧・ラベル

旧バージョンにあった「各ファイルの名前を見る」と「各ファイルにラベルをつける」を1つの表に統合しました。サイドバーの日付一覧で選んだ日付のファイルがまとめて表示されます（複数の日付を選ぶと、日付の区切り行を挟みながら全ファイルが1つの表に並びます）。表の列は以下の通りです。

- ゴミ箱アイコン（ファイル名の左端。🗑 を押すと、その行のファイルをゴミ箱へ移動します。詳しくは「3.6 ゴミ箱」を参照）
- ファイル名 / `.887` の有無 / `.csv` の有無 / 測定時間 / SWEEPS / calfact
- ラベル（数値または文字列を入力。数値ラベルは複数ファイルグラフの横軸として使えます）
- 種別（ラベルが数値・文字列のどちらとして認識されたか）
- プロットに使用（チェックを外すと、後述のグラフのファイル選択・チェックリストの対象から外れます。見出し行のチェックボックスで、表示中の全ファイルを一括でオン/オフできます）
- 共通BGとして使用（ラジオボタン。ツール全体で1つだけ指定できる、標準的なバックグラウンドファイル）
- 個別BG（プルダウン。「ファイルごとに個別のBGファイルを指定する」がONのときだけ表示されます）

**「ファイルごとに個別のBGファイルを指定する」**をONにすると、各ファイルの行に「個別BG」の選択欄が現れます。個別BGを指定した行はそのファイルを、指定していない行は引き続き共通BGを使用します（生データとバックグラウンドを1対1で対応させたい実験向け。オフの場合は既定の動作として共通BGファイル1つを全ファイルに使用します）。

### 3.3 グラフを描画する

**日付を問わず、読み込んだ全てのファイルから選択できます。** そのため、複数の日付にまたがるファイルを1つのグラフに重ね書きしたり、比較したりできます（サイドバーで選んだ日付は「2. 各ファイルの一覧・ラベル」の表示範囲にのみ影響し、グラフの対象には影響しません）。

グラフ描画エリアの幅・高さ（px）は共通設定として指定でき、PNG保存時のサイズにも反映されます。ファイル名がグラフ幅より長い場合は自動的に改行してタイトル等に表示します。5つのタブに分かれています。

**表示設定・軸範囲・質量校正の入力値は、ページを更新してもそのまま記憶されています。** 幅・高さ、「⚙ 表示設定」の内容、各タブの軸範囲、質量校正のしきい値やステップなどの入力値は、毎回入力し直す必要がありません。

幅・高さの入力欄の隣にある「⚙ 表示設定」ボタンを押すと、以下の表示条件を共通設定として変更できるパネルが開閉します（グラフサイズと同様、すべてのタブの描画・PNG保存に反映されます）。

- **データ線の色**: オンにすると、1本の線だけのグラフ（TOFスペクトル・質量スペクトル・差分など）に指定した色を使います。オフの場合は各グラフの既定色（steelblue・crimsonなど）のまま表示します。
- **カラーパレット**: 差分（Signal/BG）や複数ファイル重ね書きなど、1つのグラフに複数の線がある場合の配色です。「既定」以外（Category10・Set1・Set2・Pastel・モノトーン(青系)）を選ぶと、その配色で系列を塗り分けます。
- **線の太さ (px)**: 各グラフのデータ線の太さです。
- **凡例をグラフ内に表示する（inset）**: オフの場合は現在と同じくグラフの外側（右）に凡例を表示します。オンにするとグラフ内側（右上）に重ねて表示します。

#### (1) TOFスペクトル

選択した1ファイルの生データ（時間 vs カウント数）を表示します。測定条件の表示、グリッド表示、縦軸対数スケール、横軸・縦軸の範囲（自動/手動）を切り替えられます。

#### (2) 質量校正・質量スペクトル

TOFの時間軸を質量軸（u）に変換するための質量校正と、質量スペクトルの表示を行います。

- **ピーク検出**: 指定した検出閾値（counts）以上、かつ指定した最小ピーク間隔（channel）を満たす極大点をピークとして検出します（`scipy.signal.find_peaks` 相当のアルゴリズム）。
- **質量校正**: 検出したピークが、指定した質量ステップ（u、例: Heクラスターなら4.0026u）の整数倍間隔で並んでいると仮定し、開始質量番号 n=1〜50 の範囲で最小二乗直線回帰（`t = a√(mass) + b`）を総当たりし、決定係数(R²)が最も高い校正パラメータ `a`, `b` を採用します（自動計算）。校正ピーク検出範囲を時間軸で限定することもできます。
- 校正で得られた `a`, `b` は「差分」「複数ファイル重ね書き」タブの質量軸換算パラメータに自動反映されます。検出閾値・最小ピーク間隔・質量ステップなどの入力値自体もページ更新をまたいで記憶されるため、一度調整すれば次回以降に入力し直す必要がありません。
- **ヤコビアン補正**: 時間軸から質量軸への変換で生じる間隔の不均一さを補正するオプションです（既定でON）。
- 質量スペクトルはスムージング（移動平均）を適用したうえで表示でき、質量ステップの整数倍位置に補助線、校正に使ったピーク位置に印を表示できます。横軸(u)・縦軸の範囲指定、縦軸対数スケールに対応します。

#### (3) 差分 (Signal − BG)

SignalファイルからBackgroundファイルを差し引いた差分スペクトルを表示します。横軸は時間(ns)または質量(u)（質量校正タブのa, bパラメータを使用）を選択できます。スムージング窓幅、測定条件表示、縦軸対数スケール、軸範囲の指定に対応します。「ファイルごとに個別のBGを指定する」がONで、選んだSignalファイルに個別BGの指定がある場合、Backgroundの選択欄に自動でセットされます（手動での変更も可能です）。

#### (4) 複数ファイル重ね書き

複数の信号ファイル（数値ラベルが付与されたファイルのみ選択可）を、共通のBG（または、個別BGモードがONの場合はファイルごとに指定されたBG）を差し引いたうえで1つのグラフに重ねて表示します。**複数の日付にまたがるファイルを同時に選んで、1つのグラフとして重ね書きできます。** ラベル値の昇順に並び替えて描画されます。横軸は時間(ns)または質量(u)を選択可能、グリッド表示・縦軸対数スケール・軸範囲指定に対応します。個別BGが混在する場合は、参考用の点線BGトレースは表示されません（複数の異なるBGが使われているため）。

#### (5) 積算・比プロット

指定した2つの時間範囲（範囲1・範囲2）内のカウントをそれぞれ積算し、ファイルごとの「範囲1」「範囲2」「範囲1/範囲2の比」を計算して、ラベル値を横軸とした散布図・折れ線グラフとして表示します。結果は表としても一覧表示されます。こちらも複数の日付にまたがるファイルを同時に選択できます。個別BGモードがONの場合、ファイルごとに指定されたBGが優先して使われます（指定がなければ共通BG）。

- 対象ファイルは数値ラベルが付与されたファイルのみ選択可能です。
- 同一ラベル値を持つ複数ファイルがある場合、標準偏差などのばらつきを誤差棒として表示できます。
- データ点を線で結ぶか点のみにするか、横軸・縦軸それぞれの対数スケール表示に対応します。

### 3.4 グラフの保存・コピー

各グラフ画像の右上にカーソルを合わせると表示されるツールバー（モードバー）から、PNG画像として保存・コピーできます。

- **カメラのアイコン**: 表示中のグラフを、「⚙ 表示設定」の隣にあるグラフサイズで指定した幅・高さのPNG画像としてダウンロードします。
- **コピーのアイコン**（カメラのアイコンの隣）: 表示中のグラフをPNG画像としてクリップボードにコピーします。保存せずにそのままWord・PowerPoint・Slackなどに貼り付けて使えます。

### 3.5 コードを編集する（別ページ）

ヘッダーおよびフッターの「コードを編集する（app.js）」リンクから、別ページ（`editor.html`）が新しいタブで開きます。`app.js` の内容をブラウザ内のエディタで表示・編集し、「変更を反映して再実行」を押すとその場でプレビュー欄にツール全体を編集後のコードで再読み込みして表示できます（元の `index.html`・`app.js` はそのまま動作し続けます）。編集内容をファイルとして残したい場合は、「編集内容をapp.jsとしてダウンロード」で保存できます。

### 3.6 ゴミ箱

サイドバーの「🗑 ゴミ箱」リンクをクリックすると、別ページ（`trash.html`）に画面遷移します（新しいタブではなく、同じタブ内で移動します）。

- 「2. 各ファイルの一覧・ラベル」の表で、各行の左端にあるゴミ箱アイコン（🗑）を押すと、確認のうえでそのファイルがツール本体の一覧から取り除かれ、ゴミ箱に移動します。
- ゴミ箱ページでは、移動したファイルの一覧（ファイル名・日付・ラベル・測定時間・calfactなど）とゴミ箱に移動した日時を確認できます。
- 各ファイルには「元に戻す」（一覧に復元）と「完全に削除」（ブラウザから完全に削除。元に戻せません）のボタンがあります。「ゴミ箱を空にする」で、ゴミ箱の中身をまとめて完全に削除することもできます。
- ゴミ箱ページでの操作（復元・削除）をツール本体に反映するには、ツール本体（`index.html`）のページを再読み込みしてください。
- 「すべてクリア」を押すと、ゴミ箱の中身も含めてすべて完全に削除されます。

## 4. データの永続化とプライバシー

- 読み込んだファイルの内容（パース済みデータ）・ラベル・共通BG/個別BG指定・チェックリストの選択状態・表示設定（グラフサイズ・表示設定パネル・各タブの軸範囲・質量校正の入力値など）・サイドバーの日付選択状態・ゴミ箱の中身は、ブラウザのIndexedDBに自動保存されます。
- 次回同じブラウザで同じページを開いた際、保存内容から状態を復元します（別ブラウザ・別端末には引き継がれません）。
- セキュリティ上の配慮から、「すべてクリア」を押すと読み込んだファイルとあわせて保存されている解析結果（表示設定・校正値など）・ゴミ箱の中身もすべて削除されます。
- データが外部サーバーに送信されることはありません（Plotly.jsのCDN読み込みを除き、すべてローカルで処理されます）。

## 5. 動作環境

- モダンブラウザ（Chrome, Edge, Safari, Firefoxなど）で動作します。IndexedDBが利用できない環境では、データ保存機能なしで動作を継続します。PCでの利用を前提としています。
- インターネット接続が必要です（Plotly.jsをCDNから読み込むため）。
